One-Page Interview Cheat Sheet (Java Backend, Banking)

30s Pitch: I have 8 years in Java backend in banking. I build payment APIs on Java 21 + Spring Boot 3.2 with Oracle, MongoDB, Kafka, Docker, Kubernetes/OpenShift, and Helm. I design for idempotency, auditability, and resilience, and I operate services with Dynatrace, LogScale, and Day2Ops.

Core Flow

Client -> Akamai -> Apigee -> OpenShift Payment API -> Kafka (request)
Kafka (response) -> OpenShift Payment API -> Apigee -> Client

System Design Must-Say

  • Idempotency keys + unique constraint + TTL.
  • Outbox pattern to avoid dual writes.
  • Monotonic payment state machine.
  • SLIs/SLOs: P95 latency, error rate, Kafka lag.
  • Backpressure + DLQ + retries with jitter.
  • PCI/PII: tokenization, masking, least privilege.

Coding Must-Say

  • Two pointers / sliding window / monotonic stack.
  • Heap for top-k, quickselect trade-off.
  • BFS/DFS/Dijkstra for graphs.
  • LRU via hashmap + DLL.
  • Two heaps for streaming median.

Concurrency/Resilience

  • ConcurrentHashMap + computeIfAbsent.
  • Manual Kafka ack after successful processing.
  • Timeouts, circuit breakers, bulkheads.

One-Minute Design Template

  • Clarify requirements + scale.
  • High-level architecture.
  • Data model + APIs.
  • Consistency + reliability.
  • Observability + ops.
  • Trade-offs + evolution.
